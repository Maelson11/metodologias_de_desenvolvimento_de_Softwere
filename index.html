Desenvolvimento Orientado a Testes

O Desenvolvimento Orientado a Testes (TDD) é uma metodologia de desenvolvimento de software que coloca a ênfase na criação de testes automatizados antes da implementação do código. O TDD segue um ciclo de desenvolvimento iterativo, composto por três fases principais: "vermelho", "verde" e "refatoração".

Os três tipos de fases

1. Fase Vermelha (Red):

Objetivo: Escrever um teste automatizado que irá falhar inicialmente.

Ação: O desenvolvedor cria um teste que define a funcionalidade desejada ou especificação.

Resultado Esperado: O teste deve falhar, indicando que a funcionalidade ainda não foi implementada.

2. Fase Verde (Green):

Objetivo: Escrever o código mínimo necessário para fazer o teste passar.

Ação: O desenvolvedor implementa o código que atende aos requisitos do teste.

Resultado Esperado: O teste deve passar, indicando que a funcionalidade foi corretamente implementada.

3. Fase de Refatoração (Refactor):

Objetivo: Melhorar a estrutura do código sem alterar seu comportamento externo.

Ação: O desenvolvedor ajusta o código para torná-lo mais eficiente, legível ou modular.

Resultado Esperado: Os testes devem continuar passando após a refatoração.
